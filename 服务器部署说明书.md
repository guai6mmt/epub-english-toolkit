# EPUB 英语学习系统服务器部署说明书

这份说明书面向“你有一台自己的 Linux 服务器，希望把当前项目部署成网页应用”的场景。

推荐部署方式：

- `Docker Compose`
- `FastAPI + Uvicorn`
- 数据存储在服务器本地目录 `web_data/`

如果你只是个人使用，这是最省心的方案。

---

## 1. 当前网页版能做什么

部署后你会得到一个网页系统，支持：

- 上传 `epub`
- 后台生成学习包
- 展示今日学习内容
- 展示复习任务
- 标记完成状态
- 查看学习进度
- 查看上传任务状态
- 查看学习包详情

主程序入口：

- [run_web.py](D:/BaiduSyncdisk/英语/TE/英语学习代码/run_web.py)

Web 应用源码：

- [webapp.py](D:/BaiduSyncdisk/英语/TE/英语学习代码/src/epub_english_toolkit/webapp.py)

## 2. 服务器准备

以下说明默认你的服务器是 Ubuntu 22.04 或类似 Linux。

你需要准备：

- 一台 Linux 服务器
- 已安装 Docker
- 已安装 Docker Compose Plugin
- 一个域名可选，但强烈建议

如果还没装 Docker，可以先执行：

```bash
sudo apt update
sudo apt install -y docker.io docker-compose-plugin
sudo systemctl enable docker
sudo systemctl start docker
```

如果你当前用户没有 Docker 权限：

```bash
sudo usermod -aG docker $USER
```

然后重新登录一次。

---

## 3. 上传项目到服务器

你可以用 `git clone`，也可以手动上传。

推荐方式：

```bash
git clone <your-repo-url> epub-english-toolkit
cd epub-english-toolkit
```

如果你不是用 Git，也可以把整个项目目录上传到服务器，例如：

```bash
scp -r ./英语学习代码 user@your-server:/home/user/epub-english-toolkit
```

上传后进入项目目录：

```bash
cd /home/user/epub-english-toolkit
```

---

## 4. 配置环境变量

先复制环境变量模板：

```bash
cp .env.example .env
```

然后编辑 `.env`：

```bash
nano .env
```

建议至少修改这些字段：

```env
EPUB_TOOLKIT_PROJECT_ROOT=/app
EPUB_TOOLKIT_DATA_ROOT=/app/web_data
EPUB_TOOLKIT_DEFAULT_MODE=ielts
EPUB_TOOLKIT_DEFAULT_FOCUS_TOPICS=politics,business,culture
EPUB_TOOLKIT_WEB_USER=admin
EPUB_TOOLKIT_WEB_PASSWORD=请改成强密码
```

说明：

- `EPUB_TOOLKIT_DATA_ROOT`
  - 存放上传文件、文章库、学习包、进度数据库
- `EPUB_TOOLKIT_WEB_USER`
  - 网页登录用户名
- `EPUB_TOOLKIT_WEB_PASSWORD`
  - 网页登录密码

如果你不想启用 Basic Auth，可以把用户名和密码都留空。  
但如果服务会暴露到公网，不建议这么做。

## 5. 启动服务

在项目根目录执行：

```bash
docker compose up --build -d
```

查看运行状态：

```bash
docker compose ps
```

查看日志：

```bash
docker compose logs -f
```

默认情况下，服务监听：

- `8000` 端口

如果服务器防火墙开启了，需要放行：

```bash
sudo ufw allow 8000
```

然后你可以在浏览器里访问：

```text
http://你的服务器IP:8000
```

---

## 6. 推荐上线方式：反向代理

如果你准备长期使用，建议不要直接把 `:8000` 暴露给公网，而是加一层反向代理。

推荐：

- `Caddy`
- 或 `Nginx`

### 推荐方式：Caddy

安装 Caddy 后，可以写一个最小配置：

```caddy
your-domain.com {
    reverse_proxy 127.0.0.1:8000
}
```

这样你可以直接通过：

```text
https://your-domain.com
```

来访问系统，而且 Caddy 会自动处理 HTTPS。

## 7. 数据会存在哪里

当前 Docker Compose 已经把以下目录持久化了：

- `./web_data:/app/web_data`

也就是说，你上传的文件和学习数据会保存在服务器项目目录下的：

- `web_data/uploads`
- `web_data/library`
- `web_data/study_packs`
- `web_data/progress`
- `web_data/app.db`

这很重要，因为容器重建后数据不会丢。

你应该重点备份这个目录：

- `web_data/`

## 10. 如何真正开始学习

部署完成后，推荐这样使用。

### 第一步：打开 Upload 页面

上传一份新的 `epub`，例如一周的 `The Economist`。

你需要设置：

- `Start Date`
- `Mode`
  - 推荐 `IELTS`
- `Focus Topics`
  - 推荐：`politics,business,culture`

### 第二步：等待后台任务完成

上传后系统会创建一个后台任务。

状态可能是：

- `queued`
- `processing`
- `completed`
- `failed`

完成后会自动生成一份学习包。

### 第三步：每天只打开 Dashboard

你不需要每天重新选择文章。

每天只做三件事：

1. 打开 `Dashboard`
2. 完成今天的 `Study Task`
3. 完成今天的 `Review Task`

### 第四步：完成后点按钮

不要只“看过”。  
一定要点击 `Mark Complete`，这样系统才知道你学过了。

## 11. 推荐的实际学习方式

网页版上线后，不要把它当成“阅读器”，而要把它当成“每日学习指挥台”。

推荐你这样学：

### 每天

- 完成 1 篇主任务
- 完成 1 个复习任务
- 做 30 秒 summary
- 做 2 分钟 retell
- 写 80-120 词摘要

### 每周

- 完成一整份 study pack
- 看一次进度面板
- 回顾本周最重要的词块和观点

### 长期

- 让系统持续为你安排内容
- 不要每次自己重新决定学什么

你的学习效率会高很多，因为“决策负担”被系统替你承担了。

## 12. 当前 Web 版的边界

当前版本是一个很实用的 MVP，但不是最终版。

目前特点：

- 适合单用户
- 后台任务使用 FastAPI `BackgroundTasks`
- 数据主要保存在文件和 SQLite
- 支持上传和进度管理

当前不适合的场景：

- 多用户复杂权限
- 很高并发
- 非常大的批量队列任务

如果未来你要升级到更强版本，可以考虑：

- `PostgreSQL`
- `Celery / RQ`
- `Redis`
- 用户账户系统
- 语音上传与自动评分

---

## 13. 最推荐的下一步

部署成功后，你最应该做的不是继续堆功能，而是先形成稳定使用习惯。

建议顺序：

1. 先上传 1 本 `epub`
2. 连续使用 7 天
3. 看看你每天是否真的会打开 Dashboard
4. 再决定是否继续加：
   - 口语录音上传
   - 写作保存
   - 自动提醒
   - 每周复盘页

先让系统真的服务你的学习，再继续扩展，会更稳。
